Posted on 2025-10-27，2 min read

An environment is consist of a sequence(which means there is an order) of frames.

As we said, envronments are some memory that keeps track of the names, values, and bindings.

Frame : Keep tracking the binding between names and values. Within a frame, a name can only be bound to a value.

An environment is consist of a sequence(which means there is an order) of frames. The names have no meaning without an environment.

Now we know what the assignment statements do is changing the binding between name and value(or function) in a frame.

Looking up frames

A name evaluates to the value bound to that name in the earliest frame of the current environment in which that name is found.

The effect of an assignment statement is to bind a name to a value in the first frame of the current environment. As a consequence, assignment statements within a function body cannot affect the global frame. It's different from looking up the binding.

x = 2
def func(a,b):
    print(x)
    # x = 3   #will be ERROR if the print(x) statement is excuted
    return a+b
func(1,2)
print(x)

Rule: Within the body of a function, all the save names have to refer to the same frame.

Python doesn’t decide what x means line by line at runtime.

Instead, at functioqn definition time, the Python compiler scans the whole function body to decide which variables are local and which come from outer scopes.

If there is any assignment to a variable name anywhere in the function (like x = 3), then Python treats that name as a local variable throughout the entire function.

If you want to use the global variable x , use the statement global x to make x global.
