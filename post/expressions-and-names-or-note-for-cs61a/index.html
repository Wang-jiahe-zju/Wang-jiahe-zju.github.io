Expressions and Names | Note for CS61A

Posted on 2025-10-26，5 min read

Expressions typically describe computations.
Names refer to computational objects(values, functions).
Using the assignment statement to establish new bindings

Elements of programming

A computer program consist of instructions to either compute some value and carry out some action.
So Python code consists of expressions and statements.

  • Expressions typically describe computations.
  • Statements typically describe actions.

In programming, we deal with two kinds of elements: functions and data. Informally,

  • data is stuff that we want to manipulate
  • functions describe the rules for manipulating the data.

But soon we will discover that they are really not so dist inct.

Expressions

An expression describes a computation and evaluates to a value.

Infix Expressions

A infix expressions (or called Primitive expressions) use infix notation, where the operator (e.g., +, -, *, or /) appears in between the operands (numbers).

>>> 42
42
>>> -1 + 1
0

Operator

Numbers may be combined with mathematical operators to form compound expressions.

  • addition +
  • subtraction -
  • multiplication *
    • exponentiation **
  • division-like operators
    • Floating point division (/): divides the first number number by the second, evaluating to a number with a decimal point even if the numbers divide evenly.
    • Floor division (//): divides the first number by the second and then rounds down, evaluating to an integer.
    • Modulo (%): evaluates to the positive remainder left over from division.

Parentheses() may be used to group subexpressions together.

Call expressions

A call expression apply a function to some arguments .

>>> pow(100, 2)
10000

Anatomy of call expressions

Function notation has three principal advantages:

  • fucntions may take an arbitrary number of arguments;
  • function notation extends in a straightforward way to nested expressions;
  • Unify all the complex mathematical expressions.

Nested Expressions

Think procedurally.

To evaluate a nested call expression, Python will do the following:

  1. Evaluate the operator
  2. Evaluate operand subexpressions
  3. Apply the function that is the value of the operator subexpression to the arguments that are the values of the operand subexpressions.

For example, evaluating

>>> from operator import mul,add,sub
>>> mul(add(2, mul(4, 6)), add (3, 5))
208

Name and Assignment

Names refer to computational objects. If a value has been given a name, we say that the name binds to the value.

Using the assignment statement to establish new bindings, Names are also bound via import statements.

>>> radius = 10
>>> radius
10
>>> 2 * radius
20
>>> from math import pi
>>> pi * radius ** 2
314.1592653589793

The = symbol is called the assignment operator in Python (and many other languages). Assignment is our simplest means of abstraction.

  1. Evaluate all expressions to the right of = from left to right.

  2. Bind all names to the left of = to the resulting values in the current frame.

Names can also be bound to functions. For instance, the name max is bound to the max function we have been using. Functions are tricky to render as text, so Python prints an identifying description instead.

>>> max
<built-in function max>

In Python, names are often called variable names or variables because they can be bound to different values in the course of executing a program.

Changing the value of one name does not affect other names. See the example below, even though the name area was bound to a value defined originally in terms of radius, the value of area has not changed. Updating the value of area requires another assignment statement.

>>> radius = 10
>>> area = pi * radius ** 2
>>> area
314.1592653589793
>>> radius = 11
>>> area
314.1592653589793
>>> area = pi * radius ** 2
>>> area
380.132711084365

The possibility of binding names to values and later retrieving those values by name means that the interpreter must maintain some sort of memory that keeps track of the names, values, and bindings. This memory is called an environment.
