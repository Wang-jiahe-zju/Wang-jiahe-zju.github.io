Search fucntion

search search a value x that make the function f return True.

def search(f):
    x=0
    while 1:
        if f(x):
            return x
        x += 1

def positive(x):

Inverse function

An inverse function return g(y) such that g(f(x)) get an x.

Define inverse by using search

def inverse(f):
    """Return g(y) such that g(f(x)) → x."""
    return lambda y: search(lambda x: f(x) == y)

def square(x):
    return x * x

sqrt = inverse(square)
sqrt(256)

search(lambda x: f(x) == y) means to find an x that makes the function which take an x and return f(x) == y return a True value. In this case, f equals the inverse function 's argument square, So we will find an x makes square(x) == y evaluated as True.

lambda y: search(lambda x: f(x) == y) means a function take take function y as argument, and return an x which makes square(x) == y evaluated as True.

So that is the function sqrt, because square(sqrt(y)) == y is always True.

Be aware that is only works for the even root too.

First class elements

In general, programming languages impose restrictions on the ways in which computational elements can be manipulated. Elements with the fewest restrictions are said to have first-class status. Some of the "rights and privileges" of first-class elements are:

  1. They may be bound to names.
  2. They may be passed as arguments to functions.
  3. They may be returned as the results of functions.
  4. They may be included in data structures.

Python awards functions full first-class status, and the resulting gain in expressive power is enormous.

Funtion Decorators

Python provides special syntax to apply higher-order functions as part of executing a def statement, called a decorator.

Let's define a higher-order funtion trace which returns a function that precedes a call to its argument with a print statement that outputs the argument.

def trace(fn):
    def wrapped(x):
        print('->', fn, '(', x, ')')
        return fn
    return wrapped

@trace
def square(x):
    return x * x

square(3)

Then we use @trace to mark the def statement def square(x):, which will change the execution rule for def.

As usual, the function triple is created. However, the name square is not bound to this function.

Instead, the name square is bound to the returned function value of calling trace on the newly defined square function.

So this decorator will act equally to following code:

def square(x):
    return x * x
square = trace(square)

The decorator symbol @ may also be followed by a call expression.

  • The expression following @ is evaluated first (just as the name trace was evaluated above),
  • the defstatement second,
  • and finally the result of evaluating the decorator expression is applied to the newly defined function, and the result is bound to the name in the def statement.

A short tutorial on decorators by Ariel Ortiz gives further examples for interested students.
