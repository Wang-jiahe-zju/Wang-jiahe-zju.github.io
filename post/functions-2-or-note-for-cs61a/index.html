Designing and applying functions

Pure and Non-Pure Functions

  • Pure functions just return values.

  • Non-Pure functions have side effects.

Benefits of pure functions:

  • More reliable in compound call expressions
  • Useful in test, a list of arguments always lead to the same return value
  • Essential for writing concurrent programs (in Chapter 4)

Chapter 2 investigates a range of non-pure functions and describes their uses.

Applying function

The pricedure for calling/apllying user-defined function:

  1. Add a local frame(using the information in signature), forming a new environment
  2. Bind the function's formal parameters to its arguments in that frame
  3. Execute the body of the function in that new environment

Even in this simple example, two different environments are used.

  • The top-level expression square(-2) is evaluated in the global environment
  • The return expression mul(x, x) is evaluated in the environment created for by calling square. Both x and mul are bound in this environment, but in different frames.

design functions

Characteristics of Functions:

  • Domain: A function's domain is the set of all inputs it might possibly take as arguments.

  • Range : A function's range is the set of output values it might possibly return.

  • A pure function's behavior is the relationship it creates between input and output.

A Guide to Designing Function:

  • Give each function exactly one job.
  • Don't repeat yourself (DRY). Implement a process just once, but execute it many times.
  • Define functions generally.
